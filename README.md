# phishing-analyzer

# 🛡️ محلل التهديدات الذكي ومكافح الاحتيال (النسخة المطورة)

تطبيق أمني متطور مبني باستخدام لغة **Python** وإطار عمل **Streamlit** لفحص وتحليل النصوص والروابط المشبوهة لحماية المستخدمين من هجمات الاحتيال الإلكتروني (Phishing). يعتمد التطبيق على **3 طبقات فحص متكاملة** ويدعم ميزات التعلم الذاتي وتسهيل الوصول.

---

## 🏗️ بنية المشروع وطبقات الفحص

تم تصميم المشروع ليلبي معايير التقييم بدقة عبر تقسيم آليات الفحص إلى 3 طبقات:

1. **الطبقة الأولى (القواعد المبدئية والـ Regex):** استخراج الروابط والنطاقات (Domains) وفحص وجود كلمات الإلحاح والهندسة الاجتماعية ديناميكياً.
2. **الطبقة الثانية (السمعة الأمنية عبر API خارجي):** الاتصال بواجهة برمجية لخدمة **VirusTotal** لقراءة بيانات السمعة بصيغة JSON ومعالجة أخطاء الشبكة وحدود الطلبات (Rate Limits).
3. **الطبقة الثالثة (الذكاء الاصطناعي):** إرسال المؤشرات ونصوص البرومبت المجهزة من ملف خارجي `prompts.json` إلى نموذج **Gemini-2.5-Flash** للحصول على تحليل عميق وإخراج منظم بصيغة JSON صافية فقط.

---

## 🧠 الميزات المتقدمة المضافة

* **التعلم المستمر (Machine Learning Simulation):** يقوم النظام بتحليل ردود الذكاء الاصطناعي؛ فإذا اكتشف النموذج كلمات إلحاح أو خدعاً جديدة في النص، يتم حفظها وتحديث قاعدة المعرفة المحلية `knowledge_base.json` تلقائياً ليتعلم منها النظام في الفحوصات القادمة.
* **♿ تسهيل الوصول (Accessibility Mode):** تم توفير مفتاح وصول سريع في الواجهة، عند الضغط عليه يظهر حقل مخصص ومباشر لتحديد وفحص الرابط فوراً لمساعدة المستخدمين.

---

## 📁 تنظيم الملفات والمجلدات

المشروع منظم بالكامل في ملفات قصيرة ومحددة الوظائف:
* `app.py`: الملف الأساسي الذي يحتوي على الكود البرمجي الكامل والواجهة الرسومية وتكامل الطبقات.
* `prompts.json`: ملف خارجي معزول يحتوي على برومبت القيود وشروط الإخراج الموجه للـ LLM.
* `knowledge_base.json`: قاعدة البيانات المحلية الديناميكية لتخزين الكلمات التي يتعلمها النظام.
* `test_analyzer.py`: ملف يحتوي على اختبارات برمجية متكاملة مغطاة بأداة `pytest`.
* `requirements.txt`: ملف المكتبات والبيئة الافتراضية المطلوبة لتشغيل السيرفر.

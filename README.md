📋 1. مقدمة عن المنظومة (Project Overview)
تعد هذه المنظومة أداة أمنية سيبرانية متطورة ومحصنة، مبنية بلغة Python وواجهة Streamlit التفاعلية. تم تصميمها خصيصاً لمساعدتك ومساعدة محللي الأمن السيبراني في فحص النصوص والرسائل المشبوهة ورسائل الهندسة الاجتماعية هجمات التصيد الإلكتروني (Phishing). تعتمد المنظومة على مفهوم الدفاع الشامل المتعدد الطبقات (Multi-Layered Defense) لتوفير حكم دقيق وصارم على سلامة المحتوى والروابط المستخرجة مع حساب مؤشر رقمي دقيق لنسبة السلامة الرقمية، مما يضمن كشف التهديدات المتقدمة التي تتجاوز الفحوصات التقليدية.

🛠️ 2. المتطلبات البرمجية والاعتماديات (Prerequisites)
لكي يعمل التطبيق بكفاءة برمجية مطلقة وبدون أي أخطاء أثناء التشغيل، يجب توفير حزمة من المكتبات الرسمية المحدثة. الاعتماديات الأساسية وطريقة تثبيتها هي:

Streamlit: المكتبة الأساسية لإدارة وبناء واجهة المستخدم الرسومية التفاعلية وعرض التقارير.

Requests: لإرسال واستقبال طلبات الـ HTTP والاتصال بـ APIs الخارجية وتتبع مسارات الروابط.

Google-Genai: المكتبة الرسمية الحديثة والمعتمدة من Google للتعامل مع نماذج الذكاء الاصطناعي Gemini 2.5 بشكل مستقر وآمن.

أمر التثبيت الموحد عبر المدير الحزم:
Bash
pip install streamlit requests google-genai
🏗️ 3. البنية البرمجية وشرح الدوال بالتفصيل (Code Architecture Explained)
تتكون المنظومة من عدة دوال برمجية محصنة هندسياً تعمل بتناغم تام، وإليك شرح وظيفة ومنطق كل جزء داخل الكود:

🔹 آلية الحماية والتزامن (file_lock = threading.Lock())
الشرح: بما أن تطبيق Streamlit يعمل بنظام خيوط المعالجة المتعددة (Multi-threading) لخدمة أكثر من مستخدم في نفس الوقت، فإن محاولة تعديل ملف واحد من عدة مستخدمين بالتزامن قد تسبب تلف الملف.

المنطق الأمني: يقوم threading.Lock() بإنشاء قفل برمي يضمن أن خيطاً برمجياً واحداً فقط يمتلك صلاحية الكتابة في ملف قاعدة المعرفة في الثواني الحالية، مما يمنع ثغرات الـ Race Conditions وتلف البيانات تماماً.

🔹 دالة تحميل قاعدة المعرفة (load_knowledge_base)
الشرح: تقوم بقراءة الكلمات المفتاحية الاحتيالية (باللغتين العربية والإنجليزية) من الملف المحلي knowledge_base.json.

المنطق الأمنى: إذا لم يكن الملف موجوداً عند التشغيل الأول، تقوم الدالة تلقائياً بتحميل قائمة افتراضية ضخمة ومدمجة من كلمات الهندسة الاجتماعية الشهيرة (مثل: عاجل، حسابك مجمد، ربحت، إلخ)، مع استخدام القفل الآمن أثناء القراءة لمنع التداخل.

🔹 دالة حفظ وتحديث قاعدة المعرفة (save_knowledge_base)
الشرح: استقبال الكلمات والمصطلحات الجديدة المكتشفة ديناميكياً بواسطة الذكاء الاصطناعي وحفظها في الملف ليتعلم النظام تلقائياً مع الوقت.

المنطق البرمجي: تقوم بتحويل القائمة إلى set لإزالة أي تكرار للمصطلحات، ثم تعيد تحويلها إلى قائمة وتكتبها داخل بيئة محمية بـ with file_lock.

🔹 دالة كشف وتتبع الروابط (unshorten_url)
الشرح: مواجهة أساليب المهاجمين الذين يستخدمون خدمات اختصار الروابط (مثل Bitly أو TinyURL) لإخفاء المواقع الخبيثة.

المنطق الأمني والمحاكاة: تقوم بالدخول إلى الرابط وتتبع قفزات إعادة التوجيه (Redirects) حتى 3 مستويات عبر طلبات من نوع requests.head خفيفة الوزن. كما تم تزويدها بـ User-Agent يحاكي متصفح Google Chrome حقيقي لتفادي الحظر أو التضليل من خوادم الأعداء، بالإضافة إلى ضبط مهلة انتظار timeout=2.5 منعاً لتجميد السكربت.

🔹 دالة استخراج المؤشرات (extract_indicators)
الشرح: تمثل الطبقة الأولى للفحص؛ حيث تستخرج الروابط عبر التعبيرات النمطية (RegEx) وتستخلص النطاقات (Domains) وتتحقق من كلمات الإلحاح.

التحصين ضد الالتفاف (Whitelist Subdomain Bypass Protection): الكود التقليدي كان يمكن خداعه بنطاق مثل google.com.attacker.com لأنه ينتهي بكلمة موثوقة. تم تعديل المنطق هنا ليفحص النطاق كاملاً ويكون التطابق صارماً بحيث يشمل النطاق الرئيسي أو نطاقاً فرعياً حقيقياً يسبقه نقطة الفصل . (مثل sub.google.com).

🔹 دالة فحص السمعة الدولية (check_url_reputation)
الشرح: تمثل الطبقة الثانية؛ حيث تأخذ النطاقات غير الموجودة في القائمة البيضاء وترسلها إلى محرك VirusTotal عبر الـ API الخاص به لمعرفة ما إذا كان النطاق مسجلاً كـ Malicious أو Phishing في قواعد البيانات العالمية.

🔹 دالة التحليل السياقي بالذكاء الاصطناعي (analyze_with_llm)
الشرح: الطبقة الثالثة والأعمق؛ حيث ترسل كافة المؤشرات المستخرجة إلى نموذج Gemini 2.5 Flash. لا ينظر النموذج للروابط فقط بل يحلل الأسلوب (هل هناك إلحاح؟ تهرب؟ انتحال صفة بنك؟).

الميزة الذكية: يقوم النموذج باستخراج كلمات احتيالية جديدة لم تكن مسجلة في النظام من قبل ويعيدها لتحديث قاعدة البيانات المحلية.

🔹 دالة اتخاذ القرار النهائي وحساب النتيجة (make_final_decision)
الشرح: دمج نتائج الفحص الثلاثي وتوليد النتيجة النهائية:

Safe (آمن): إذا كان في القائمة البيضاء أو لم يعثر الذكاء الاصطناعي أو VirusTotal على تهديدات (النتيجة الرقمية: 85% إلى 98%).

Suspicious (مشبوه): إذا احتوى النص على كلمات إلحاح وهندسة اجتماعية قوية (النتيجة الرقمية: 50%).

Dangerous (خطير جداً): إذا أكد VirusTotal أو Gemini وجود تهديد أو احتيال مباشر (النتيجة الرقمية: 3%).

🔐 4. إعداد متغيرات البيئة السرية (Environment Setup)
لحماية مفاتيح الـ API الخاصة بك من التسريب داخل الكود، تعتمد المنظومة على قراءتها من متغيرات البيئة بالنظام. يجب ضبطها كالآتي قبل التشغيل:

على أنظمة Windows (Command Prompt):
DOS
set GEMINI_API_KEY="your_gemini_api_key_here"
set VIRUSTOTAL_API_KEY="your_virustotal_api_key_here"
على أنظمة Linux / macOS (Terminal):
Bash
export GEMINI_API_KEY="your_gemini_api_key_here"
export VIRUSTOTAL_API_KEY="your_virustotal_api_key_here"
🚀 5. طريقة تشغيل المنظومة (Deployment & Running)
بعد الانتهاء من إعداد البيئة وتثبيت المكتبات، توجه إلى المجلد الذي يحتوي على ملف الكود (مثلاً app.py) ونفذ الأمر التالي:

Bash
streamlit run app.py
سيقوم خادم Streamlit بالانطلاق فوراً وفتح واجهة المستخدم الرسومية عبر المتصفح على الرابط الافتراضي: http://localhost:8501.

♿ 6. أدوات تيسير الوصول والميزات المساعدة (Accessibility Tools)
المنظومة مزودة بمركز مساعدة مدمج يدعم أساليب متطورة لراحة المستخدم:

المتحدث الصوتي التلقائي: ينطق نتيجة التقرير والحيثيات الأمنية بصوت عالي الجودة متوافق مع اللغة العربية ar-SA عبر متصفح المستخدم دون الحاجة لمكتبات ثقيلة وخارجية.

المترجم السيبراني: يقوم بترجمة الحيثيات الأمنية والتحليلات الصادرة باللغة العربية إلى لغة إنجليزية سيبرانية احترافية فوراً باستخدام الـ LLM لتقديمها في التقارير الرسمية.

📄 7. تخصيص التوجيهات عبر ملف prompts.json (اختياري)
إذا كنت ترغب في تغيير طريقة تحليل نموذج Gemini أو تشديد القيود، يمكنك إنشاء ملف اختياري باسم prompts.json بجانب الملف الرئيسي وتضمين الهيكل التالي داخله:

JSON
{
    "phishing_analysis_prompt": "قم بتحليل مؤشرات الفحص التالية بدقة واستخرج قراراً أمنياً صارماً، مع ضرورة إعادة الإجابة بصيغة JSON تحتوي على المفاتيح status و reason و discovered_urgency_keywords. المؤشرات الممررة هي: {indicators}"
}
